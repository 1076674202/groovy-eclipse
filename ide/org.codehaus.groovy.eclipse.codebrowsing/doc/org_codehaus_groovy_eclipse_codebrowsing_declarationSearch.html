Groovy Eclipse Code Browsing

Identifier:
org.codehaus.goovy.eclipse.codebrowsing.codebrowsing

Since:
[Enter the first release in which this extension point appears.]

Description:
An IDeclarationSearchContext determines the current declaration search context. Leaving this blank chooses the global context. Enter a unique string to identify a custom context. For example, 'org.grails.context' identifies a grails context that is active only when editing a file in a Grail application. An example is a Grails declaration search processor looking for controllers for a view.

Configuration Markup:

<!ELEMENT extension (searchProcessor+ , searchContext+)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED

>


<!ELEMENT searchContext EMPTY>

<!ATTLIST searchContext

name      CDATA #REQUIRED

contextId CDATA #REQUIRED

class     CDATA #REQUIRED

>

An IDeclarationSearchContext defines when certain IDeclarationSearchProcessor intstances are active.


  • name - A translatable name suitable for display.
  • contextId - The id of the context, e.g. 'org.grails'
  • class - The class implementing the search context.

<!ELEMENT searchProcessor EMPTY>

<!ATTLIST searchProcessor

name         CDATA #REQUIRED

contextId    CDATA #REQUIRED

astNodeClass CDATA #REQUIRED

class        CDATA #REQUIRED

>

An IDeclarationSearchProcessor searches for the declarations of some ASTNode.


  • name - A translatable name suitable for display.
  • contextId - The context the processor operates in, blank chooses the default Groovy context.
  • astNodeClass - The ASTNode class to be passed to the processor.
  • class - The class implementing the search processor.
