ITypeEvaluationContext

The type evaluation context provides all the state necessary for the type evaluator to evaluate some expression. In general is not implemented by clients. It would be implemented in special cases for example, a console bound to the debugger state to allow for inspecting the live object using Groovy expressions.

Type evaluation classes and interfaces are found in the package org.codehaus.eclipse.core.types.

The interface:

public interface ITypeEvaluationContext {
    ClassLoader getClassLoader();
    String[] getImports();
    Type lookupSymbol(String name);
    Field lookupField(String type, String name, boolean accessible, boolean staticAccess);
    Property lookupProperty(String type, String name, boolean accessible, boolean staticAccess);
    Method lookupMethod(String type, String name, String[] paramTypes, boolean accessible, boolean staticAccess);
}

When getting a context for some source code, instances of the interface are retrieved using a builder.
Example:

ITypeEvaluationContext context = new TypeEvaluationContextBuilder()
        .sourceCodeContext(context)
        .symbolTable(table)
        .memberLookup(lookup)
        .classLoader(project.getProjectClassLoader())
        .imports(imports)
        .done();

Type evaluation is described in its own section.
