Source Code and Evaluation Contexts

Code completion, browsing and type inference all require some kind of contexts to function in. There are two types of contexts in GroovyEclipse represented as follows:

  • The interface org.codehaus.eclipse.core.context.ISourceCodeContext to identify a location in source code. This interface is not designed to be extended, default implementations are returned by the SourceCodeContextFactory.
  • The interface org.codehaus.eclipse.core.types.ITypeEvaluationContext, which contains state information for type evaluation.
