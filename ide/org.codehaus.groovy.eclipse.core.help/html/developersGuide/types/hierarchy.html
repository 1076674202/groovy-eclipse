The Type Hiearchy

The type hiearchy is described, with excerpts of the class fields and methods where appropriate. See the JavaDoc for details.

The classes in the Type hiearchy can be found in package org.codehaus.groovy.eclipse.core.types
package org.codehaus.groovy.eclipse.core.types;

Type

The root of all Type classes.

public abstract class Type implements Modifiers, Comparable {
    public static final int CLASS = 1;
    public static final int MEMBER = 2;
    public static final int LOCAL_VARIABLE = 3;
    public static final int PARAMETER = 4;
    public static final int FIELD = 5;
    public static final int METHOD = 6;
    protected String signature;
    protected String name;
    protected int modifiers;
    private boolean inferred;
    public boolean isGroovyType() { ... }
    public boolean isInferred() { ... }
}

The signature is simply the same as that returned by methods in Class. e.g. 'java.lang.String' or '[Ljava.lang.String;' or '[Z' or 'byte' and so on.
Other points of interest here are the two methods isGroovyType() and isInferred() . It it very important that these be implemented in the correct way. Failure to do so can confuse the type evaluator.

isGroovyType()

This method should return false if the type is a Java type. Java types are rarely used. If required they are type names prefixed with 'Java'.

isInferred()

This method should return false if the type is known. This is true for Java types. It is also true in most cases when extensions are implemented to handle dynamic types.

Derived Types

The derived types act mainly as holders of information relating to their hierarchy. For example, Member types have a reference to their declaring class. For details see the JavaDoc.

Utilities

Class TypeUtil contains various utilities for creating types amongst other things.
