Evaluation Type Hierarchy

There is a small type hiearchy that consists of the lowest level objects found in GroovyEclipse. This hiearchy is encountered time and again and it's use is required when extending GroovyEclipse with type information for dynamic types, such as Grails dynamic methods.
It is not to be confused with the IJavaElement hiearchy found in the JDT.

The hiearchy:

  • Type
    • ClassType
    • LocalVariable
    • Parameter
    • Member
      • Field
      • Property
      • Method
The root of this hiearchy, Type , contains a type signature and is marked as either class or instance. In the case of Method, the signature represents the return type of the method.
