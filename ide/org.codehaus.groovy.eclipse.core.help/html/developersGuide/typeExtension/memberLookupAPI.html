The IMemberLookup Interface

This is the heart of GroovyEclipse type evaluator. Default implementations exist to lookup members of different types that are found in the project class path. The interface is quite simple, but implementation difficulty varies depending on the application.

public interface IMemberLookup extends Modifiers {
    public Field[] lookupFields(String type, String name, boolean accessible, boolean staticAccess, boolean exact);
    public Property[] lookupProperties(String type, String name, boolean accessible, boolean staticAccess, boolean exact);
    public Method[] lookupMethods(String type, String name, boolean accessible, boolean staticAccess, boolean exact);
    public Method[] lookupMethods(String type, String name, String[] paramTypes, boolean accessible,
            boolean staticAccess, boolean exact);
}

A brief explanation:
  • type - the name of a class.
  • name - the name of what to lookup.
  • accessible - if true, ony members which are visible from the type are looked up. For example, super class private members are not accessible.
  • static access - if true, then only static members should be returned.
  • exact - if true, exact matches only. If false, then the name is to be used as a prefix.
Once this interface is implemented by a plug-in and specified in the plug-in manifest, it will automatically be used by GroovyEclipse for code completion, type evaluation and more.
